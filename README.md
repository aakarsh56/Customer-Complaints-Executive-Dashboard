# Customer-Complaints-Executive-Dashboard
Business Objective
The goal of this project was to transform raw financial complaint data into actionable insights for executive leadership. By creating a 3-tier reporting system, I enabled stakeholders to move from high-level KPIs to specific root-cause analysis in under three clicks.

🚀 Key Features & Layout
Page 1: Executive Overview
Goal: Monitor overall health and volume.

Features: High-level cards showing Total Complaints, Timely Response %, and MoM Change.

Visuals: A monthly trend area chart and a custom-branded navigation bar.

Page 2: Trends & Geography
Goal: Identify "Where" and "How" complaints are happening.

Features: Interactive Map visual showing hotspots (CA & FL) and a Donut chart breaking down submission methods (Web vs. Phone).

Navigation: Custom horizontal buttons for seamless page switching.

Page 3: Deep Dive (Root Cause)
Goal: Explain the "Why" behind the data.

Features: Decomposition Tree used to drill down from Product to specific Issue.

UX Highlight: A custom Summary Ribbon providing a written synthesis of findings.

🛠️ Technical Stack & Skills
Power BI Desktop: Advanced report authoring and UI/UX design.

DAX (Data Analysis Expressions): * Calculated measures for Month-over-Month (MoM) % Change.

Dynamic Report Page Tooltips for on-hover insights.

Data Transformation: Cleaned and structured data to ensure accuracy across synchronized slicers.

UI/UX: Implemented a cohesive Navy (#001F3F) theme with custom slanted buttons and professional branding.

💡 Top 3 Business Insights
Product Friction: Credit Cards are the primary driver, making up 33% of the total volume.

Key Bottleneck: "Problem with a purchase" is the most frequent specific issue reported.

Channel Efficiency: Over 70% of complaints come via the Web, suggesting a successful digital-first intake strategy.

📂 How to Use This Repo
Download the .pbix file from the /Dashboard folder.

Open in Power BI Desktop.

Use the Overview/Trends/Deep Dive buttons to navigate the report.
